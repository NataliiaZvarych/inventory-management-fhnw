# 📦 Inventory Management System – FHNW Projekt

Dieses Projekt wurde im Rahmen des Moduls *Objektorientierte Programmierung* im Studiengang **Wirtsachtsinformatik** an der FHNW entwickelt.
Ziel des Projekts ist die Konzeption und Umsetzung eines strukturierten Backend-Systems unter Einsatz moderner Python-Technologien, klarer Architekturprinzipien sowie einer ORM-basierten Datenbankanbindung.

---

## 🎯 Zielsetzung

Im Mittelpunkt steht die Entwicklung einer zuverlässigen und wartbaren Lösung zur Verwaltung von Lagerbeständen, die insbesondere auf die Anforderungen kleiner und mittelständischer Unternehmen ausgerichtet ist.

---

## ⚙️ Anwendungsanforderungen

### 🧩 Problemstellung

In vielen kleinen Unternehmen erfolgt die Lagerverwaltung noch manuell oder mithilfe unzureichender Softwarelösungen.
Dies führt häufig zu folgenden Herausforderungen:

* Inkonsistente und fehleranfällige Datenbestände
* Eingeschränkte Transparenz über aktuelle Lagerbestände
* Unzureichende Nachvollziehbarkeit von Lagerbewegungen

Um diesen Problemen entgegenzuwirken, ist ein strukturiertes System erforderlich, das eine zuverlässige Speicherung, Verwaltung und Auswertung der Daten ermöglicht.

---

### 📌 Szenario

Die Anwendung richtet sich an Mitarbeitende, die für die Verwaltung von Lagerbeständen verantwortlich sind.
Sie ermöglicht die strukturierte Erfassung und Verwaltung von:

* Produkten
* Kategorien
* Lagerorten
* Lagerbewegungen

Alle Informationen werden in einer relationalen Datenbank gespeichert, wodurch eine hohe Datenkonsistenz sowie eine lückenlose Nachvollziehbarkeit gewährleistet werden.
